Stage: DNL InfoSec Portal

Van 16 september tot 13 december heb ik 13 weken stage gelopen bij dotNET lab. Dit is een IT consultancy bedrijf gelegen in Heultje. Ze hebben meer dan 20 jaar ervaring en zijn met meer dan 60 gecertificeerde Microsoft professionals.

De stageopdracht die ik kreeg, was het maken van een Engelse webapplicatie waar klanten hun security scores op konden zien, het DNL InfoSec Portal. Een werknemer van dotNET lab zou dan een security audit doen bij de klant. Bij deze audit wordt het bedrijf op 18 CIS controls (Center for Internet Security) getest.

Deze controls zijn eigenlijk 18 vereisten waaraan een bedrijf moet voldoen om een veilig bedrijf te zijn. De scores van de audit werden dan omgezet naar één score die, inclusief opgestelde verbeterpunten, werd ingegeven in de webapplicatie. De klant kon op zijn beurt de score en verbeterpunten terugvinden op het portaal en zijn beveiliging verbeteren.

Wat was het doel van de stage?

Het doel van mijn stage bij dotNET lab was ervoor zorgen dat ik klaar was voor de start van mijn professionele carrière. Hiermee bedoel ik dat ik zowel mijn hard- en softskills moest verbeteren.

Daarom dat ik als full stack developer aan de slag kon gaan met het project. Tijdens dit projectwerk heb ik zowel mijn technische, als mijn persoonlijke vaardigheden ontwikkeld. Een echte werkomgeving was toch een stuk anders dan de omgeving op school.

Dit project gaf me een gevoel van verantwoordelijkheid, aangezien ik de enige full stack developer was. Ook de mogelijkheid om aan mezelf te werken was enorm belangrijk tijdens de stageperiode. Vandaar dat ik enorm blij ben dat ik mijn theoretische kennis heb kunnen toepassen in een echte werkomgeving.

Hoe heb ik dit doel bereikt?

Dit heb ik gedaan door alles stap voor stap aan te pakken. Voor ik kon starten aan de webapplicatie zelf moest ik eerst een grondige analyse doen van het project.

Deze analyse was nodig omdat er in de toekomst nog ging verder gewerkt worden aan de applicatie. De werknemer die eraan zou verder werken kon dan mijn analyse doornemen en alles te weten komen over het doel van het project en hoe dit allemaal gemaakt moest worden. Gedurende een zestal weken was de analyse geëvolueerd tot een gedetailleerde analyse waar alles in stond beschreven. Hierin had ik voor elk onderdeel van de applicatie uitgelegd wat dit onderdeel was, waarom dit nodig was en de stappen om dit te implementeren. Door het maken van deze analyse heb ik enorm veel bijgeleerd over het analyseren van een project en het belang ervan.

Toen ik aan de applicatie kon starten heb ik volledig vanaf nul alles opgezet. Dit wil zeggen dat ik zowel de back-end als de front-end had gemaakt. Bij de back-end was het belangrijk dat ik alles opzette met behulp van clean architecture, dit met de nieuwste versie van .NET, .NET 5.0. Voor de front-end had ik dan weer verschillende componenten aangemaakt om op deze manier de applicatie zo overzichtelijk mogelijk te houden. Ook dit gebeurde in de nieuwste versie van Angular, Angular 18.
