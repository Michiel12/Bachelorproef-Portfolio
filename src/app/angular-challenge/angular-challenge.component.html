Angular challenge: Trip planner

Samen met nog 4 andere medestudenten kregen we de opdracht om een trip planner te maken, vandaar dat wij als groep TravelSync hebben gemaakt.

Opdracht

Voor het vak Angular kregen we de opdracht om samen in een groep van vijf studenten een trip planner te maken, de Angular challenge. Dit was een echte challenge, we namen het namelijk op tegen 10 andere groepen. Elke groep moest een applicatie maken doorheen het eerste semester van het derde jaar en deze applicatie dan voorstellen voor de jury.

Deze challenge gebeurt jaarlijks in samenwerking met dotNET lab, waarbij twee van hun experten ons wekelijks kwamen helpen waar nodig. Zo keken ze elke week onze code na zodat we op het einde van het traject een mooi resultaat hadden.

Voor deze challenge hebben we een mooie overzichtelijke dashboard gemaakt waar de gebruiker trips kan beheren. Deze trip bestond uit verschillende dagen, bestemmingen en activiteiten. Dit was allemaal onderdeel van het project waar we enorm hard voor hadden gewerkt. Dat was dan ook te zien aan het resultaat, want mijn groep en ik hadden de eerste plaats behaald.

Mijn bijdrage

In dit project had ik rol van full stack developer op mij genomen. Dit wil zeggen dat ik zowel back-end als front-end heb gemaakt. Een onderdeel wat dan weer vooral mijn verantwoordelijkheid was, was het implementeren van de Ticketmaster API.

Ik was met het idee gekomen om festivals/optredens vanuit onze applicatie te vinden. Hiervoor heb ik de Ticketmaster API gebruikt om alle optredens op een bepaalde locatie, tussen specifieke datums te tonen. Op deze manier kon de gebruiker makkelijk en snel kijken welke evenementen er waren op het moment dat ze op die locatie waren.

Dit was in het begin best lastig, aangezien ik dit nog niet had gebruikt. Na wat onderzoek doen en dingen uitproberen kwam ik erachter wat er allemaal mogelijk was met die API. Hierdoor heb ik een handige tool kunnen toevoegen aan onze applicatie.
